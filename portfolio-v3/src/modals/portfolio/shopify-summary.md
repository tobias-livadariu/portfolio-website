# summary.md

Shopify gave me a bigger-system version of the work I already liked: product surfaces, data correctness, and infrastructure all tied together.

I worked on the marketing analytics team, where the code had to support merchants at real scale. I fixed marketing page crashes affecting about 150 shops per day, helped decouple admin reporting pages from an expensive legacy data pipeline, and worked through database migration problems where small test failures pointed to much larger production-data risks.

The strongest part of the internship was learning how to reason about safety. A lot of the work was careful and deliberate: reading query patterns, writing rollout notes, splitting changes into reviewable PRs, and explaining risks clearly before touching shared systems. That is the kind of engineering I want to keep doing.

I also built Stickify during hackdays with Waterloo classmates, a React and Vite tool that turns Slack threads into structured tickets. We pitched it to Shopify executives, and my mentor estimated it could save about 30 minutes per day for a regular user.

The result is that I am more comfortable with ambiguity than I was before Shopify. I can move across frontend, backend, data, and infrastructure work, and I know how to slow down when correctness matters.
